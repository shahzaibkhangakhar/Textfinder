Project Overview

A sophisticated question-answering system that combines document retrieval with AI-powered answer generation. The system processes PDF documents, creates semantic embeddings, and generates accurate answers based on retrieved context.

Key Features

  • Multi-format document processing (PDF, TXT, MD)
  • Advanced text chunking with semantic preservation
  • FAISS-based vector similarity search
  • Language model-powered answer generation
  • RESTful API with FastAPI backend
  • React-based web interface
  • Batch processing capabilities

Core Components

1. Document Retriever

The retriever component is responsible for processing documents and finding relevant information.

Key Features

  • Smart Text Chunking: Uses RecursiveCharacterTextSplitter for intelligent document segmentation
  • Vector Embeddings: Converts text chunks into high-dimensional vectors using sentence transformers
  • FAISS Indexing: Efficient similarity search using Facebook's FAISS library
  • Metadata Tracking: Maintains source information and processing timestamps

2. Answer Generator

The generator component processes retrieved context and generates accurate answers.

Key Features

  • Structured Prompts: Carefully crafted prompts for consistent output
  • Context Integration: Incorporates retrieved chunks into generation process
  • Batch Processing: Efficient handling of multiple queries
  • Response Formatting: Structured and readable answer generation

3. RAG Pipeline

The pipeline orchestrates the entire workflow from question to answer.

Data Flow

  1. User submits question
  2. Retriever finds relevant document chunks
  3. Generator creates structured prompt
  4. Language model generates answer
  5. Response is formatted and returned

Deployment Guide

Prerequisites

  • Python: 3.9 or higher
  • Memory: Minimum 4GB RAM
  • Storage: 2GB free space
  • Network: Internet access for model downloads

Local Development

# Clone repository
git clone <repository-url>
cd NLProc-Proj-M

# Install dependencies
pip install -r requirements.txt

# Run development server
python -m uvicorn api.main:app --host 127.0.0.1 --port 8000 --reload

Performance & Optimization

Chunk Size Optimization

  • Academic papers: chunk_size=256, overlap=50
  • News articles: chunk_size=512, overlap=100
  • Technical docs: chunk_size=384, overlap=75

FAISS Index Selection

  • Small datasets: IndexFlatL2
  • Medium datasets: IndexIVFFlat
  • Large datasets: IndexIVFPQ

Troubleshooting

Common Issues

  • Dependency Conflicts: Use compatible versions from requirements-prod.txt
  • Memory Issues: Reduce chunk size or use smaller embedding models
  • Model Download Failures: Check internet connection and model availability
  • FAISS Index Errors: Rebuild index or use smaller index types
